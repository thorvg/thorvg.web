The following is a quick start to show you how to use the essential APIs.\n\nFirst, you should initialize the ThorVG engine:\n\n```typescript\nimport ThorVG from '@thorvg/webcanvas';\n\nconst TVG = await ThorVG.init({ renderer: 'gl' }); // WebGL renderer\n```\n\nThen it would be best if you prepared an empty canvas for drawing on it:\n\n```typescript\nconst canvas = new TVG.Canvas('#canvas', {\n width: 800,\n height: 600\n});\n```\n\nNext you can draw multiple shapes on the canvas:\n\n```typescript\nconst rect = new TVG.Shape(); // generate a shape\nrect.appendRect(50, 50, 200, 200, { rx: 20, ry: 20 }); // define it as a rounded rectangle (x, y, w, h, rx, ry)\nrect.fill(100, 100, 100); // set its color (r, g, b)\ncanvas.add(rect); // add the rectangle to the canvas\n\nconst circle = new TVG.Shape(); // generate a shape\ncircle.appendCircle(400, 400, 100, 100); // define it as a circle (cx, cy, rx, ry)\n\nconst fill = new TVG.RadialGradient(400, 400, 150); // generate a radial gradient (cx, cy, radius)\n\nfill.setStops( // set the gradient colors info\n [0.0, [255, 255, 255, 255]], // 1st color values (offset, [r, g, b, a])\n [1.0, [0, 0, 0, 255]] // 2nd color values (offset, [r, g, b, a])\n);\n\ncircle.fill(fill); // set the circle fill\ncanvas.add(circle); // add the circle to the canvas\n```\n\nThis code generates the following result:\n\n

\n\nYou can also draw your own shapes and use dashed stroking:\n\n```typescript\nconst path = new TVG.Shape(); // generate a path\npath.moveTo(199, 34); // set sequential path coordinates\npath.lineTo(253, 143);\npath.lineTo(374, 160);\npath.lineTo(287, 244);\npath.lineTo(307, 365);\npath.lineTo(199, 309);\npath.lineTo(97, 365);\npath.lineTo(112, 245);\npath.lineTo(26, 161);\npath.lineTo(146, 143);\npath.close();\n\npath.fill(150, 150, 255); // path color\n\npath.stroke({\n width: 3, // stroke width\n color: [0, 0, 255, 255], // stroke color (r, g, b, a)\n cap: TVG.StrokeCap.Round, // stroke cap style\n join: TVG.StrokeJoin.Round, // stroke join style\n dash: [10, 10] // stroke dash pattern (line, gap)\n});\n\ncanvas.add(path); // add the path to the canvas\n```\n\nThe code generates the following result:\n\n

\n\n## Render Backends\n\nThorVG WebCanvas supports both WebGL and the next-generation WebGPU, optimized for modern browsers and high-performance rendering pipelines. Designed to empower developers with cutting-edge graphics capabilities.\n\n| Backend | Browser Support |\n|---------|-----------------|\n| **WebGL (gl)** | Chrome/Firefox/Safari 90+ |\n| **WebGPU (wg)** | Chrome/Edge 113+/Firefox 141+/Safari 26+ |\n\n### Backend-Specific Initialization\n\n```typescript\n// WebGL renderer\nconst TVG = await ThorVG.init({ renderer: 'gl' });\n\n// WebGPU renderer (requires async init)\nconst TVG = await ThorVG.init({ renderer: 'wg' });\n```\n\n[Back to contents](#contents)\n
\n\n## Memory Management\n\nWebCanvas provides automatic memory management through FinalizationRegistry, but you can also manage memory explicitly:\n\n```typescript\n// Automatic cleanup when GC runs\nconst shape = new TVG.Shape();\ncanvas.add(shape);\nshape = null; // Call dispose()\n\n// Automatic cleanup on page unload (registry.ts)\nwindow.addEventListener('beforeunload', () => {\n if (hasModule()) {\n const Module = getModule();\n Module.term(); // Terminate WASM\n }\n});\n\n// Explicit cleanup (recommended for predictable memory management)\nshape.dispose();\npicture.dispose();\nanimation.dispose();\n\n// Terminate WASM module (call this when done)\nTVG.term();\n```\n\n[Back to contents](#contents)\n
